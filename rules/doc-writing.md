# 문서 작성 원칙 — ansible-practice

## 언어
- 문서(README, CLAUDE.md)는 **한국어**로 작성
- Ansible 모듈명, 변수명, 파일명은 영어 그대로
- 태스크 `name:`은 한국어로 동작을 명확히 서술

## 롤 README 구조
1. **역할** — 설치/설정 대상 명확히
2. **지원 환경** — OS, 배포판, 버전
3. **주요 변수** — 표 형식 (변수명 | 기본값 | 설명)
4. **의존 롤** — meta/main.yml 기준
5. **태그** — 부분 실행 가능한 태그 목록
6. **실행 예시** — `--check` 드라이런 포함
7. **트러블슈팅** — 증상/원인/해결 형식

## 변수 문서화
- 모든 변수에 inline 주석: `# 한국어 설명`
- Vault 변수는 `vault_` 접두사 + 반드시 README에 목록화
- 민감한 기본값은 `""` 또는 `CHANGEME`으로 표시

## 코드 주석 패턴

```yaml
# ===============================================================
# {섹션 제목}
# {부가 설명}
# ===============================================================
- name: "{서비스명} 패키지 설치"
  package:
    name: "{{ package_name }}"
    state: present
```

## 주의사항 표시
- prod 전용 설정: `# prod 전용` 주석
- 위험한 태스크: README에 `> **주의**:` 경고 블록
